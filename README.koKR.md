# ModbusSerialServerApp

이 문서는 Modbus Serial Server 애플리케이션의 기능과 사용법에 대한 설명입니다.

## 개발 중인 프로젝트
이 프로젝트는 개발중에 있습니다. 사용중 발생할 수 있는 모든 책임은 사용자에게 있습니다.

## 개요
Modbus Serial Server 는 RTU 및 ASCII 프로토콜을 모두 지원하는 Modbus 직렬 서버를 구현한 Python 기반 GUI 애플리케이션입니다. 직관적인 인터페이스를 통해 Modbus 데이터 블록을 실시간으로 모니터링하고 관리할 수 있습니다.

## ✨ 주요 특징
###🔌 프로토콜 지원
  * RTU 모드 : 효율적인 통신을 위한 바이너리 프로토콜 형식
  * ASCII 모드 : 디버깅이 용이한 텍스트 기반 프로토콜 형식
  * 다중 슬레이브 환경을 위한 구성 가능한 슬레이브 ID

### 📊 데이터 관리
  * 4개의 Modbus 데이터 블록 :
    * Coils - 1비트 출력 읽기/쓰기
    * Discrete Input - 읽기 전용 1비트 입력
    * Input Register - 읽기 전용 16비트 레지스터
    * Holding Register - 16비트 레지스터 읽기/쓰기
  * 데이터 블록당 최대 65,535개의 레지스터 지원
  * 레지스터 값의 실시간 폴링 및 표시(500ms 간격)

### 🎨 사용자 인터페이스
  * 3탭 인터페이스 :
    1. 데이터 블록 - 주소 열이 있는 가상 스크롤 표시
    2. 설정 - 시리얼 포트 구성 패널
    3. 시리얼 로그 - 실시간 수신/송신 메시지 모니터링
  * TX/RX LED 표시등 - 직렬 통신에 대한 시각적 피드백
  * 프로그램 로그 - 타임스탬프가 포함된 오류 및 정보 메시지

### 🔧 설정 옵션
  * 시리얼 포트 설정 :
    * 자동 감지 기능을 이용한 포트 선택
    * 전송 속도: 1200, 2400, 4800, 9600, 19200, 38400, 57600, 115200 bps
    * 데이터 비트: 7 또는 8
    * 패리티 비트: 없음, 짝수, 홀수
    * 정지 비트: 1, 1.5 또는 2
    * 슬레이브 ID: 장치별로 구성 가능

### 🚀 성능 최적화
  * 가상 스크롤링 : 보이는 행만 렌더링됩니다(6,554개 이상의 행을 효율적으로 지원).
  * 비동기 처리 : asyncio와 스레딩을 사용한 비차단 UI
  * 선택적 폴링 : 오버헤드를 줄이기 위해 보이는 레지스터 범위만 폴링합니다.

### 📡 로깅 및 디버깅
  * 색상으로 구분된 시리얼 로그(수신은 파란색, 송신은 주황색)
  * 밀리초 단위 정밀도의 타임스탬프
  * 데이터 수신/전송 시 LED 깜빡임으로 피드백 제공
  * 활성 연결 매개변수를 포함한 시스템 상태 표시

## 기술 세부 정보
### 구조
  * 주요 구성 요소 : ModbusServerApp클래스(Tkinter 기반 GUI)
  * 스레딩 모델 : 비동기 I/O를 위한 별도의 이벤트 루프 스레드
  * 통신 : 직렬 전송을 사용하는 PyModbus 라이브러리
  * GUI 프레임워크 : Tkinter와 ttk 위젯

### 종속성
   ```
   - pymodbus >= 3.0.0
   - pyserial
   - tkinter (usually included with Python)
   ```

### Modbus 표준
  * Modbus 직렬 통신 사양을 따릅니다.
  * RTU 모드에 대한 CRC 오류 검사
  * ASCII 모드에 대한 LRC 오류 검사
  * 모든 표준 기능 코드에 대한 읽기/쓰기 작업 지원

## 📦 설치
1. 저장소 복제
   ```bash
   git clone https://github.com/nsicko42/ModbusSerialServerApp.git
   cd ModbusSerialServerApp
   ```
2. 종속성 설치
   ```bash
   pip install pymodbus>=3.0.0 pyserial
   ```
3. 애플리케이션 실행
   ```bash
   python modbus_server.py
   ```

## 사용법
### 서버 시작
1. 애플리케이션을 실행합니다:
   ```bash
   python modbus_server.py
   ```
2. 설정 탭으로 이동, 시리얼 포트 설정
3. \[ ▶ \] 버튼을 클릭해 서버 시작

### 서버 중지
\[ ■ \] 버튼을 클릭해 서버 중지

## 시스템 상태 표시
|요소|의미|
|----|----|
|상태 표시줄|현재 서버 상태 및 연결 매개변수를 표시합니다.|
|정보 텍스트|타임스탬프가 포함된 실시간 애플리케이션 메시지|

## 💡 팁 & 트릭
  * 포트 감지 : "포트 새로 고침" 버튼을 사용하여 새로 연결된 직렬 장치를 동적으로 감지합니다.
  * 성능 : 레지스터 범위가 넓은 경우, 가상 스크롤링 기능으로 부드러운 UI 반응성을 보장합니다.
  * 디버깅 : 통신 문제를 해결하려면 시리얼 로그 탭을 활성화하세요.
  * 다중 장치 지원 : 동일한 직렬 회선에서 여러 장치를 지원하도록 슬레이브 ID를 변경합니다.
